## üñºÔ∏è OG Image Generation (Server-Side)

Using `nuxt-og-image` with Satori renderer for server-side OG image generation:

- **Route**: `/_og-image/[path]`
- **Component**: `DefaultOgImage.vue`
- **Renderer**: Satori (SVG-based)
- **Size**: 1200x630px
- **Fonts**: System fonts (to avoid font download issues)

### OG Image Component Location
`app/components/OgImage/DefaultOgImage.vue`

### How It Works
1. Each page uses `defineOgImage()` in the `usePageSeo()` composable
2. OG images are generated on-demand when pages are accessed
3. Images are cached for performance
4. Dynamic OG images are created with page-specific titles and descriptions

## üîß Troubleshooting

### OG Images Not Generating
- Check that `nuxt-og-image` is in modules array
- Verify the OG Image component exists at `app/components/OgImage/DefaultOgImage.vue`
- Check server logs for errors
- Ensure Node.js version is compatible (v18+)

### Font Download Issues
- Using system fonts in OG image component to avoid network requests
- Alternative: Download fonts locally and use `path` option

### Build Errors
- Run `npm run postinstall` to regenerate types
- Delete `.nuxt` and `node_modules/.cache` if needed
- Check for TypeScript errors in useSeo composable
